# Stage 8：セキュリティ & 運用

**目安：第 29〜31 週（各週 約 10 時間）**

Stage 8 では、**セキュリティ**と**運用**の実務を学びます。OWASP の再現演習、RBAC 設計、脅威モデリング、構造化ログ、アラート設計、ポストモーテムを通じて、インシデント対応（レポート・切り分け）ができることが目標です。

- **入り口**: [Stage 7：AWS & IaC](/stage7)（第 25〜28 週）を修了していること。インフラをコードで管理し、環境を再現できるレベルを想定しています。まだの場合は、先に Stage 7 の教材を完了させてから始めてください。

## 出口（修了時にできること）

- **インシデント対応**ができる（レポートを書き、切り分けの流れを説明できる）
- **OWASP** を意識し、SQL インジェクション・XSS 等の対策を説明できる
- **RBAC** と脅威モデリングの基礎を理解している
- **構造化ログ・アラート・ポストモーテム**の役割を説明できる

---

## Stage 8 修了チェック

Stage 8 を修了したかどうかは、次のチェックリストで確認できます。すべて「はい」なら修了です。

- [ ] OWASP の代表的な脆弱性（SQL インジェクション・XSS 等）を再現し、対策を説明できる
- [ ] RBAC を設計し、役割と権限の対応を説明できる
- [ ] 脅威モデリングの基礎（STRIDE 等）を説明できる
- [ ] 構造化ログとアラート設計の考え方を説明できる
- [ ] ポストモーテムの目的と記載項目を説明できる
- [ ] 疑似障害の切り分けレポートを書ける

評価は [ルーブリック](/rubric) の「Stage 8：セキュリティ・運用」に従います（OWASP 対応・脅威モデル・ログ・アラート）。
