# Stage 2：AI 統合開発基盤

**目安：第 4〜6 週（各週 約 10 時間）**

Stage 2 では、**Git** によるバージョン管理、**Pull Request（PR）** を使った開発の流れ、そして**要件・受入基準・AI 利用ポリシー**の考え方を学びます。PR ベースで開発できることと、AI に任せてよい範囲・人間が検証する領域を理解することが目標です。

- **入り口**: [Stage 1](/stage1) を修了していること。Stage 1 で作った CLI ツールを、この Stage で Git に push する想定です。まだ Stage 1 が終わっていない場合は、先に Stage 1 の第 3 週まで完了させてから始めてください。

---

---

## 出口（修了時にできること）

- **PR ベースで開発できる**（feature ブランチで変更し、PR を作成してマージする流れを体験している）
- **要件→受入基準→境界設計**ができる（小さな機能単位で要件を書き、Given-When-Then で受入基準を書ける）
- **AI 利用ポリシー・禁止領域を理解している**（AI に任せてよい範囲と、必ず人間が検証する領域を説明できる）

---

## Stage 2 修了チェック

Stage 2 を修了したかどうかは、次のチェックリストで確認できます。すべて「はい」なら修了です。

- [ ] リモートリポジトリを clone できる
- [ ] 変更を add ・commit ・push できる
- [ ] ブランチを作成し、merge できる
- [ ] rebase ・squash の意味を説明できる
- [ ] PR を 1 本作成し、マージまで体験している
- [ ] 受入基準を Given-When-Then 形式で書ける
- [ ] AI に任せてよい範囲と、人間が検証する領域を説明できる

評価は [ルーブリック](/rubric) の「Stage 2：設計・AI 統合」に従います（要件・受入基準・AI ポリシーが明確か）。

---

## 提出物の目安

- **第 5 週**: 実際に 1 本 PR を作成し、マージまで行う。または PR テンプレートと DoD（Definition of Done）を記載したドキュメント
- **第 6 週**: 受入基準の例（Given-When-Then）を 1 件以上書いたドキュメント、AI 利用ポリシー（チーム or 個人用）のテンプレートまたは記述例
