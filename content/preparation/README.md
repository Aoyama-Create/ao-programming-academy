## 1. 準備物・PC の条件

### 必要なもの

- **自分専用で使える PC**（Windows / macOS / Linux のいずれか）  
  プログラミングでは、キーボードでコードを入力し、ターミナル（後述）でコマンドを打ち、ファイルを保存する操作が基本になります。スマートフォンやタブレットだけでは、これらの操作が難しく、本カリキュラムの演習を進められないため、PC が必要です。

- **安定したインターネット接続**  
  ソフトのダウンロードや、ドキュメント・教材の閲覧に使います。

### OS について

この教材では、**Windows** と **macOS** を主に想定しています。手順が違うところは、OS ごとに見出しを分けて書いてあります。Linux を使う場合は、多くの操作が macOS と同様です。

### スペックの目安

- **メモリ**: 4GB 以上（8GB あると快適です）
- **ストレージ**: 空き容量 5GB 以上
- 最近 5 年程度の一般的なスペックの PC であれば、十分です。

### 学校の PC や共有 PC を使う場合

学校の PC や共有 PC を使う場合は、**ソフトのインストール（管理者権限）** ができるかどうか確認してください。インストールが禁止されている環境では、Node.js や Cursor を入れられない場合があります。また、Cursor の AI 機能を使うには**アカウント作成（サインイン）**が必要なので、外部サービスへのサインインが制限されている環境では、そこも確認が必要です。いずれも難しい場合は、先生や管理者に相談するか、自宅の PC で進める必要があります。

## 2. 最低限の用語（用語集）

プログラミングの教材を読むときに、よく出てくる用語をまとめました。初めて見る言葉は、ここで確認できます。

| 用語 | 説明 |
|------|------|
| **プログラミング** | コンピュータに「こう動け」と指示を書くこと。その指示の集まりが**プログラム**です。 |
| **コード / ソースコード** | プログラムを書いた文字（テキスト）のこと。プログラミング言語の文法に従って書きます。 |
| **実行（する）** | 書いたプログラムを、コンピュータに実際に動かすこと。 |
| **ファイル** | データやコードを保存した「1 つのまとまり」です。名前と**拡張子**（例：`main.js`）があります。 |
| **拡張子** | ファイル名の末尾にある「.（ピリオド）＋英数字」の部分。例：`main.js` の `.js` は「JavaScript のコードです」という意味です。 |
| **フォルダ（ディレクトリ）** | ファイルを整理して入れておく「入れ物」です。フォルダの中に、さらにフォルダを入れることもできます。 |
| **パス** | ファイルやフォルダの「場所」を表す書き方。例：`Documents/myapp/main.js` は「Documents フォルダの中の myapp フォルダの中の main.js というファイル」を指します。 |
| **ターミナル（コマンドライン）** | 文字だけでコンピュータに指示を出す画面です。キーボードで**コマンド**（命令）を打って操作します。 |
| **コマンド** | ターミナルに打ち込む「命令」のこと。例：`cd`（フォルダを移動）、`dir` / `ls`（ファイル一覧を表示）。 |
| **カレントディレクトリ** | ターミナルで「今いるフォルダ」のこと。コマンドは、この「今いるフォルダ」を基準に実行されます。 |
| **エディタ** | コードを書くためのソフト。メモ帳と違って、コード用の色付け（シンタックスハイライト）や補完が使えます。最近は、AI による補完やチャットを備えたエディタもあります（本カリキュラムで使う **Cursor** がそれです）。 |
| **インストール** | ソフトを PC に導入して、使えるようにすること。 |
| **ブラウザ** | ウェブページを見るソフト（Chrome、Edge、Safari など）。 |
| **ダウンロード** | インターネットからファイルを PC に保存すること。 |
| **解凍** | 圧縮されたファイル（.zip など）を、中身を取り出して使える形にすること。 |
| **AI コード補完** | 書きかけのコードの続きを、AI が予測して提示してくれる機能。提示された内容を採用するかどうかは、自分で判断します。 |
| **AI チャット（アシスタント）** | エディタの中で AI に質問したり、コードの作成を依頼したりできる機能。生成されたコードは、必ず自分で読んで理解してから使います。 |

### 3.1 エディタの準備（Cursor）

#### なぜエディタを使うか

コードは「テキスト（文字）」で書きます。メモ帳でも書けますが、**プログラミング用のエディタ**を使うと、コードに色が付いたり、入力の補助が効いたり、フォルダごと開けて管理しやすくなります。本カリキュラムでは **Cursor（カーソル）** を使います。理由は次の 3 つです。

1. **VS Code をベースに作られたエディタ**です。画面の見た目・操作・拡張機能は VS Code とほぼ同じなので、インターネット上にある「VS Code の使い方」の記事や動画は、そのまま Cursor でも役に立ちます。
2. **AI による補完やチャットが最初から組み込まれています**。本カリキュラムは Stage 2 以降で「AI をどう使い、どこは人間が検証するか」を扱うため、最初から AI を使えるエディタで学び始めます。
3. **無料のプランがあり**、Windows / macOS / Linux のどれでも使えます。

#### インストール手順

**Windows の場合**

1. ブラウザで [https://cursor.com/](https://cursor.com/) を開く
2. 「Download」のボタンをクリックし、インストーラー（.exe）をダウンロードする
3. ダウンロードしたファイルをダブルクリックして実行する
4. 画面の指示に従って進める（既定のままで問題ありません）
5. インストールが終わると、Cursor が起動する

**macOS の場合**

1. ブラウザで [https://cursor.com/](https://cursor.com/) を開く
2. 「Download」のボタンをクリックし、.zip または .dmg をダウンロードする
3. ダウンロードしたファイルを開き、Cursor をアプリケーションフォルダにドラッグしてコピーする
4. アプリケーションから「Cursor」を起動する

> **macOS で「開発元を確認できないため開けません」と出た場合**: Cursor のアイコンを右クリック（または Control キーを押しながらクリック）して「開く」を選び、確認画面でもう一度「開く」を押すと起動できます。

（ここに画像：Cursor のダウンロードページの画面）

#### 初回起動時の設定

1. 初回起動時に、キーバインド（キー操作の割り当て）やテーマ（配色）を選ぶ画面が出ます。**すべて既定のままで問題ありません**。「VS Code から設定をインポートしますか？」と聞かれたら、VS Code を使ったことがない人はスキップして構いません
2. **アカウント作成（サインイン）** … AI 機能を使うにはサインインが必要です。画面の案内に従ってアカウントを作成します。無料のプランで始められます
3. 日本語で使いたい場合は、左のアイコンから「拡張機能」を開き、「Japanese」で検索して「Japanese Language Pack」をインストールする。再起動するとメニューが日本語になります（Cursor は VS Code がベースなので、VS Code 向けの拡張機能がそのまま使えます）
4. 「ファイル」→「フォルダーを開く」から、これからコードを置くフォルダ（例：`Documents/study-js`）を選んで開く。何もないフォルダを新規作成して開いてもかまいません

（ここに画像：Cursor でフォルダを開いた画面）

#### Cursor の AI 機能の基本

Cursor には多くの AI 機能がありますが、最初は次の 3 つだけ知っていれば十分です。

| 機能 | 呼び出し方 | 何ができるか |
|------|------------|--------------|
| **Tab 補完** | コードを書いていると、続きの候補がグレーの文字で表示される。`Tab` キーで採用、`Esc` キーで却下 | 書きかけのコードの続きを AI が予測して提示する |
| **インライン編集** | `Cmd + K`（Windows は `Ctrl + K`） | 選択した箇所について「この関数に説明のコメントを付けて」のように日本語で指示し、書き換えてもらう |
| **チャット** | `Cmd + L`（Windows は `Ctrl + L`） | 「このコードは何をしている？」「このエラーの原因は？」のように、AI に質問する |

#### 最初のうちの使い方（大切です）

AI は強力ですが、**使い方を間違えると学習が進みません**。次の 3 点を意識してください。

- **生成されたコードは、必ず自分で読む**。「なぜこう書けるのか」を自分の言葉で説明できる状態にしてから採用してください。説明できないコードを、そのまま残さないようにします。
- Stage 1 の目的は、**文法と考え方を自分の頭に入れること**です。そのため最初のうちは、「答えを丸ごと書かせる」使い方よりも、**「分からない箇所を質問する」使い方**から始めるのがおすすめです。エラーが出たときにチャットで原因を聞く、書いたコードの意味を確認する、といった使い方です。
- 「AI に任せてよい範囲」と「必ず人間が検証すべき範囲」の線引きは、**Stage 2 第 6 週**で「AI 利用ポリシー」として自分で文書化します。今の段階では、上の 2 点を守っておけば大丈夫です。

### 3.2 ターミナルの「存在」と基本操作

#### ターミナルとは

**ターミナル**（Windows では「コマンドプロンプト」や「PowerShell」、macOS では「ターミナル」）は、文字だけでコンピュータに指示を出す画面です。プログラミングでは、プログラムを**実行**したり、フォルダを移動したり、ファイルの一覧を確認したりするときに使います。最初はとっつきにくく感じるかもしれませんが、よく使うコマンドは少ないので、少しずつ慣れていきましょう。

#### ターミナルの開き方

- **Windows**: スタートメニューで「cmd」または「PowerShell」と入力して、表示された「コマンドプロンプト」または「Windows PowerShell」をクリック
- **macOS**: Spotlight（Command + スペース）で「ターミナル」と入力して Enter。または「アプリケーション」→「ユーティリティ」→「ターミナル」を開く

（ここに画像：ターミナルを開いた画面）

#### 最低限の操作

| やりたいこと | Windows（PowerShell） | macOS / Linux |
|--------------|------------------------|---------------|
| 今いるフォルダを確認する | `pwd` | `pwd` |
| フォルダを移動する | `cd フォルダ名` | `cd フォルダ名` |
| ファイル一覧を表示する | `dir` | `ls` |

- **カレントディレクトリ**: 今、ターミナルが「いる」フォルダのことです。`pwd` でその場所が表示されます。
- **コマンド**: 上のように、ターミナルに打ち込む命令のことです。

#### やってみよう

1. ターミナルを開く
2. `pwd` と打って Enter を押す → 今いるフォルダのパスが表示される
3. `cd Documents` のように、どこかフォルダ名を入れて Enter → そのフォルダに移動する
4. `dir`（Windows）または `ls`（macOS）と打って Enter → そのフォルダの中身（ファイル・フォルダの一覧）が表示される

これができれば、ターミナルの「入口」はクリアです。Stage 1 の第 1 週では、ここで開いたフォルダに `.js` ファイルを置いて、`node ファイル名.js` で実行する練習をします。

### 3.3 Node.js のインストール

#### Node.js とは

**Node.js** は、JavaScript を **PC 上で実行するためのソフト**です。普段、JavaScript はブラウザの中で動きますが、Node.js を入れると、ターミナルから「`node ファイル名.js`」と打って、自分で書いた JavaScript を実行できるようになります。本カリキュラムの Stage 1 では、この Node.js を使ってプログラムを動かします。

#### インストール手順（LTS 版を推奨）

**Windows の場合**

1. ブラウザで [https://nodejs.org/](https://nodejs.org/) を開く
2. 「LTS」（Long Term Support）と書かれた方のボタンをクリックし、インストーラー（.msi）をダウンロードする
3. ダウンロードしたファイルをダブルクリックして実行する
4. 画面の指示に従い、「Next」で進め、最後に「Install」を選ぶ
5. インストールが終わったら、**ターミナル（PowerShell やコマンドプロンプト）を一度閉じて、もう一度開き直す**

**macOS の場合**

1. ブラウザで [https://nodejs.org/](https://nodejs.org/) を開く
2. 「LTS」と書かれた方のボタンをクリックし、.pkg をダウンロードする
3. ダウンロードした .pkg を開き、画面の指示に従ってインストールする
4. インストールが終わったら、**ターミナルを一度閉じて、もう一度開き直す**

（ここに画像：Node.js のダウンロードページの画面）

#### インストールの確認

ターミナルで、次のコマンドを打って Enter を押します。

```text
node --version
```

バージョン番号（例：`v20.10.0`）が表示されれば、インストールは成功です。

**バージョンが表示されない場合**

- ターミナルを**閉じて、もう一度開き直して**から、もう一度 `node --version` を試してください（インストール直後は「パス」が反映されていないことがあります）
- それでも表示されない場合は、インストール手順をやり直すか、公式サイトの「インストールガイド」を確認してください
