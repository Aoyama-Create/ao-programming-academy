# 第 27 週：CI で terraform plan、本番模擬デプロイ（提出物）

**目安時間**: 約 10 時間（解説・手順・演習）

---

## 学習目標

この週が終わると、次のことができるようになります。

- **CI** で `terraform plan` を実行し、PR 時にインフラの変更内容をレビューできる
- **本番模擬デプロイ**の手順を説明し、apply のタイミングと承認フローを設計できる
- 環境再現を「10 分」程度で行う手順を文書化できる

---

## 1. CI で terraform plan

PR のたびに `terraform plan` を実行し、差分をコメントやアーティファクトとして残します。本番への `apply` は手動または承認付きで行い、plan の結果を必ず確認してから適用する流れにします。

（ここに GitHub Actions 等での plan 実行・出力の見せ方・apply の条件を追記できます。）

---

## 2. 提出物の目安

- **CI で terraform plan**: PR 時に plan が実行され、変更内容が確認できる状態にすること
- **本番模擬デプロイ**: stg または本番相当環境へのデプロイ手順、あるいはそのレポート。再現にかかる時間の目安（10 分を目指す）も含めるとよい

評価は [ルーブリック](/rubric) の「Stage 7：運用」を参照してください。
