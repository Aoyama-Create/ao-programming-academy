# Stage 5：データ設計

**目安：第 18〜20 週（各週 約 10 時間）**

Stage 5 では、**データベースの設計とパフォーマンス**を学びます。インデックス、EXPLAIN、N+1 対策、分離レベル、Redis キャッシュを通じて、遅い原因を説明し、計測に基づいて改善できることが目標です。

- **入り口**: [Stage 4：バックエンド（Laravel）](/stage4)（第 13〜17 週）を修了していること。Stage 4 で作った API と DB を前提に、データ層の設計・性能を扱います。まだの場合は、先に Stage 4 の教材を完了させてから始めてください。

## 出口（修了時にできること）

- **遅い原因を EXPLAIN で説明できる**（インデックスの有無・N+1 の有無を説明できる）
- **分離レベル**とキャッシュのトレードオフを理解している
- **Before/After 計測**に基づいて改善理由を説明できる

---

## Stage 5 修了チェック

Stage 5 を修了したかどうかは、次のチェックリストで確認できます。すべて「はい」なら修了です。

- [ ] インデックスの役割を説明し、必要なカラムに追加できる
- [ ] EXPLAIN で実行計画を確認し、遅い理由を説明できる
- [ ] N+1 問題を理解し、対策（eager load 等）を適用できる
- [ ] 分離レベルと Redis キャッシュの基礎を説明できる
- [ ] 計測レポートで改善前後を説明できる

評価は [ルーブリック](/rubric) の「Stage 5：設計・品質」に従います（インデックス・EXPLAIN・計測レポート）。
