# 第 14 週：認証

**目安時間**: 約 10 時間（解説・手順・演習）

---

## 認証と認可の違い

**認証**（Authentication）は「誰であるか」を識別することです。ログイン・セッションや API トークンによって、リクエストしてきたのがどのユーザーかを確定します。この週では認証に集中します。

**認可**（Authorization）は、そのユーザーが「何をしてよいか」を制御することです。Policy や RBAC で操作の可否を判定します。認可は第 15 週で扱います。**認証 → 認可** の順で学ぶことで、まず「誰か」を識別し、そのうえで「何をしてよいか」を制御する流れが明確になります。

---

## 学習目標

この週が終わると、次のことができるようになります。

- **Laravel Breeze** または **Sanctum** でログイン・ログアウト・セッション管理を実装できる
- **API トークン**（Sanctum）で、SPA や外部クライアントからの認証付きリクエストを扱える
- 認証済みユーザーを前提にした API や画面の設計ができる

## 3. 認証と次の週（認可）への接続

認証ができると、コントローラで `auth()->user()` により「いま誰がリクエストしているか」が分かります。第 15 週では、この「認証済みユーザー」を前提に、Policy で「そのユーザーはこの操作をしてよいか」という認可を実装します。
