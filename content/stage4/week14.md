# 第 14 週：Policy・認可、API 設計、OpenAPI 生成

**目安時間**: 約 10 時間（解説・手順・演習）

---

## 学習目標

この週が終わると、次のことができるようになります。

- **Policy** で「誰が何をしてよいか」を定義し、コントローラで適用できる
- **API 設計**の考え方（リソース・HTTP メソッド・ステータスコード）を説明できる
- **OpenAPI**（Swagger）で API 仕様を記述・生成し、ドキュメント化できる

---

## 1. Policy と認可

Laravel の **Policy** は、特定のモデルに対する「このユーザーはこのアクションを実行できるか」を判定するクラスです。`authorize()` や `Gate` と組み合わせて、コントローラで認可チェックを行います。

## 2. API 設計

REST 的な API では、URL をリソース、HTTP メソッドを操作（GET=取得、POST=作成、PUT/PATCH=更新、DELETE=削除）に対応づけます。ステータスコード（200, 201, 400, 401, 404 など）で結果を表現します。

## 3. OpenAPI 生成
