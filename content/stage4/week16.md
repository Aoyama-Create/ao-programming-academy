# 第 16 週：承認フロー付き受注 API・非同期メール・監査ログの課題、ER 図・シーケンス図・トランザクション設計理由（提出物）

**目安時間**: 約 10 時間（解説・手順・演習）

---

## 学習目標

この週が終わると、次のことができるようになります。

- **承認フロー**を持つ受注 API を設計・実装できる（状態遷移・認可の考慮）
- **非同期メール**と**監査ログ**を組み合わせた処理を実装できる
- **ER 図・シーケンス図・トランザクション設計理由**を提出物として説明できる

---

## 1. 承認フロー付き受注 API

受注（オーダー）に「下書き」「申請中」「承認済み」「却下」などの状態を持たせ、状態に応じた操作（申請・承認・却下）と認可（誰がどの状態を変更できるか）を設計します。

## 2. 非同期メールと監査ログ

承認や受注確定時にメールをキューで送信し、同時に「誰がいつ何をしたか」を監査ログテーブルに記録します。トランザクションの境界（ログ記録とメール投入を同じトランザクションにするかどうか）を意識して設計します。

## 3. 提出物の目安

- **ER 図**: 主要なエンティティとリレーション
- **シーケンス図**: 代表的なフロー（例：受注申請〜承認）のシーケンス
- **トランザクション設計理由**: どこを 1 トランザクションにしたか、その理由を簡潔に説明
