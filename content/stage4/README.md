# Stage 4：バックエンド（Laravel）

**目安：第 13〜16 週（各週 約 10 時間）**

Stage 4 では、**Laravel** を使ってバックエンド API を設計・実装する力を身につけます。MVC・ルーティング、Eloquent・マイグレーション、認可（Policy）、API 設計、トランザクション・Queue・Mail まで一通り体験し、トランザクション設計・認可設計ができることが目標です。

- **入り口**: [Stage 3：フロント実務](/stage3)（第 9〜12 週）を修了していること。Stage 3 で作ったフロントが、この Stage で作る API のクライアントとしてつながる想定です。まだの場合は、先に Stage 3 の教材を完了させてから始めてください。

## 出口（修了時にできること）

- **トランザクション設計・認可設計ができる**（境界を意識し、一貫性と権限を適切に設計できる）
- **API 設計**ができる（ルーティング・リソース・OpenAPI の流れを体験している）
- **非同期処理・監査**ができる（Queue、Mail、監査ログを扱える）

---

## Stage 4 修了チェック

Stage 4 を修了したかどうかは、次のチェックリストで確認できます。すべて「はい」なら修了です。

- [ ] MVC に沿ってルーティング・モデル・コントローラを配置できる
- [ ] Eloquent とマイグレーションでモデルと DB を管理できる
- [ ] Policy で認可を設計し、API で適用できる
- [ ] トランザクションの境界を説明し、Queue・Mail を利用できる
- [ ] ER 図・シーケンス図・トランザクション設計理由を説明できる

評価は [ルーブリック](/rubric) の「Stage 4：設計・品質・セキュリティ」に従います（ER・シーケンス・認可・トランザクション）。
