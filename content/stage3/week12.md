# 第 12 週：管理画面 CRUD・複雑フォームの課題、コンポーネント設計書・型設計説明（提出物）

**目安時間**: 約 10 時間（課題に取り組む・提出物を書く）

---

## 学習目標

この週が終わると、次のことができるようになります。

- **一覧・詳細・作成・更新・削除**の流れを 1 つの管理画面で実装できる
- **複雑なフォーム**（複数フィールド・項目の追加・削除）をコンポーネント分割で扱える
- **コンポーネント設計書**と**型設計の説明**を提出物として書ける

---

## 1. 課題の位置づけ

第 9〜11 週で学んだ TypeScript・React・フォーム・API 連携を総合し、**管理画面の CRUD** と **複雑フォーム** を 1 つのアプリで実装する課題です。バックエンドはまだないため、**モック API** または **MSW** でデータを用意してください。

---

## 2. 課題仕様の例

### 対象データ

例：**商品（Item）** の管理

- **一覧**: 商品のリストを表示する。各行に「詳細」「編集」「削除」の操作を用意する。
- **詳細**: 1 件の商品の内容を表示する。「編集へ」リンクを置いてよい。
- **新規作成**: 商品を追加するフォーム。必須項目（名前・価格など）と任意項目を設ける。
- **編集**: 既存の商品を更新するフォーム。詳細で表示したデータを初期値として表示する。
- **削除**: 削除してよいか確認し、OK なら削除する（モックでは state やモック API 内のデータから削除）。

### 複雑フォームの例

「項目の追加・削除ができるフォーム」を 1 つ含めると、コンポーネント分割の練習になります。

- 例：商品に「タグ」を複数持たせる。タグを 1 つずつ入力し、「追加」でリストに加え、「削除」でリストから外す。
- この部分を **TagInput** のような子コンポーネントに分け、親から `value: string[]` と `onChange` を渡して扱う形にするとよいです。

### 技術的な制約

- React + TypeScript で実装する。
- データの永続化は不要（ページをリロードするとモックの初期状態に戻ってよい）。モック API や useState で「メモリ上」の CRUD でよい。
- ルーティングは、React Router などで「一覧」「詳細」「新規」「編集」のパスを分けても、1 ページ内で state で表示を切り替えてもよいです。

---

## 3. 実装のステップ案

1. **型の定義**  
   Item（および必要なら Tag など）の interface を決める。API の「レスポンス形」「リクエスト形」も型で定義する。
2. **モック API**  
   一覧取得・1 件取得・作成・更新・削除の関数を用意する。中身はメモリ上の配列でよい。
3. **一覧画面**  
   第 11 週と同様、型付きで一覧を表示し、各行から「詳細」「編集」「削除」に進めるようにする。
4. **詳細画面**  
   id で 1 件取得し、表示する。
5. **新規作成フォーム**  
   第 10 週のフォームの考え方で、バリデーションとエラー表示を付ける。送信時はモック API の「作成」を呼ぶ。
6. **編集フォーム**  
   詳細で取得したデータを初期値にしたフォーム。送信時はモック API の「更新」を呼ぶ。
7. **削除**  
   確認ダイアログまたは確認用画面を出し、OK ならモック API の「削除」を呼ぶ。
8. **複雑フォーム部分**  
   タグの追加・削除など、必要に応じて子コンポーネントに分ける。

---

## 4. 評価の観点（ルーブリックとの対応）

[ルーブリック](/rubric) の Stage 3「コンポーネント設計・型設計・エラー網羅」に沿って、次の点を意識してください。

- **コンポーネント設計**: どのコンポーネントが何を受け持ちかが分かるように分けているか。一覧・詳細・フォーム・複雑な入力部分をコンポーネントに分割しているか。
- **型設計**: 主要なデータ（Item など）と API の入出力が interface で定義され、画面間で一貫して使われているか。
- **エラー網羅**: フォームのバリデーション、API 呼び出しの loading / error を扱っているか。削除の確認など、操作の取り消しができるか。

## 5. 提出物

次の 2 つを提出してください。

### コンポーネント設計書

- **どのコンポーネントが何を受け持っているか**を短くまとめたもの。
- 例：「ItemList：一覧表示。props は items と onEdit, onDelete。」「ItemForm：新規・編集兼用。props は initialData（編集時）と onSubmit。」

### 型設計の説明

- **主要な interface**（Item, 作成用の型, API のレスポンス型など）を挙げ、それぞれが「何のための型か」を 1〜2 文で説明する。
