# Stage 3：フロント実務（React + TS）

**目安：第 9〜12 週（各週 約 10 時間）**

Stage 3 では、**TypeScript** と **React** を使って、仕様変更に強い UI を設計する力を身につけます。型付きのコンポーネント、フォームとバリデーション、API 連携（モック）、そして管理画面の CRUD まで一通り体験することが目標です。

- **入り口**: [Web フロント基礎](/weekly#web-フロント基礎htmlcssjs)（第 7〜8 週）を修了していること。HTML/CSS/JavaScript で静的なページと軽いインタラクションが書けるレベルを想定しています。まだの場合は、先に第 7〜8 週の教材を完了させてから始めてください。

## 出口（修了時にできること）

- **仕様変更に強い UI を設計できる**（コンポーネントの責務を分け、型で一貫性を保てる）
- **型駆動開発**ができる（型定義を先に書き、実装を型に沿わせる流れを体験している）
- **API 連携・エラーハンドリング**ができる（フロントから API を呼び、loading/error を扱える）

---

## Stage 3 修了チェック

Stage 3 を修了したかどうかは、次のチェックリストで確認できます。すべて「はい」なら修了です。

- [ ] TypeScript の型・interface を使ってコンポーネントを書ける
- [ ] フォームの状態管理とバリデーション・エラー表示ができる
- [ ] API（またはモック）を型付きで呼び出し、一覧・詳細を表示できる
- [ ] 管理画面 CRUD をコンポーネント分割で実装している
- [ ] コンポーネント設計・型設計を説明できる

評価は [ルーブリック](/rubric) の「Stage 3：設計・品質・AI 統合」に従います（コンポーネント設計・型設計・エラー網羅）。
