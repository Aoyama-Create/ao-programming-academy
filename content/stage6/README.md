# Stage 6：品質保証

**目安：第 21〜23 週（各週 約 10 時間）**

Stage 6 では、**テストと CI** で品質を担保する力を身につけます。Unit テスト（PHPUnit / Jest）、Integration テスト、CI でのテスト強制、静的解析（PHPStan/ESLint）を通じて、テストで壊れない状態を作ることが目標です。

- **入り口**: [Stage 4](/stage4) または [Stage 5](/stage5) で実装したコードがあること。テストを書く対象となるアプリケーションが前提です。まだの場合は、先に Stage 4 の教材を完了させてから始めてください。

## 出口（修了時にできること）

- **テストで壊れない状態を作れる**（主要なパスをテストでカバーし、CI で通す）
- **静的解析**でコードの品質を機械的にチェックできる
- **カバレッジ**を意識し、70% 以上を目指せる

---

## Stage 6 修了チェック

Stage 6 を修了したかどうかは、次のチェックリストで確認できます。すべて「はい」なら修了です。

- [ ] Unit テスト（PHPUnit または Jest）を書き、実行できる
- [ ] Integration テストで API や DB を伴うパスをテストできる
- [ ] CI でテストを実行し、失敗時にマージを防げる
- [ ] 静的解析（PHPStan/ESLint）を導入し、指摘を解消できる
- [ ] テストカバレッジ 70% 以上を目指している

評価は [ルーブリック](/rubric) の「Stage 6：品質」に従います（テスト・CI・カバレッジ）。
